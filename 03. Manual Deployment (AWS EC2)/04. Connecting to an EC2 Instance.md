# üß© **6. Connect to the Instance via SSH**

You connect from your local machine (terminal) to the EC2 instance.

SSH = Secure Shell
A protocol used to securely access remote machines via terminal.

### üñ• For macOS & Linux

SSH works **out of the box** in the terminal.

### ü™ü For Windows

Windows does *not* include SSH by default (unless using PowerShell/OpenSSH on newer versions), so you must:

**Option 1:** Install **WSL2**, which gives you a Linux shell
**Option 2:** Use **PuTTY** (a GUI SSH client)

---

# üß© **7. Using the AWS ‚ÄúConnect‚Äù Button**

In the EC2 dashboard:

* Select your instance
* Click **Connect**
* Choose **SSH Client**

AWS gives you:

* A `chmod` command (Linux/macOS/WSL only)
* An SSH command to connect, similar to:

```
chmod 400 example-1.pem
ssh -i example-1.pem ec2-user@YOUR_INSTANCE_PUBLIC_IP
```

### Windows + PuTTY users

‚ö†Ô∏è Skip the `chmod` step.
PuTTY provides a GUI to load the key file.
AWS also links to their detailed PuTTY guide.

---

# üß© **8. Connect**

* On first connection, type `yes` to confirm.
